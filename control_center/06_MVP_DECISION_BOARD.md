## 读题后填写

正式赛题已解析，详见：`control_center/17_PROBLEM_ANALYSIS_MEDICAL_KNOWLEDGE_AGENT.md`

### 必做 MVP

必须完成，否则不能构成可提交作品。

- [ ] PDF/MD/TXT 上传与解析；PDF 必须稳定，能显示章节结构。
- [ ] 单本教材知识点抽取和知识图谱展示。
- [ ] 2 本以上教材的跨教材合并决策，包含 `merge / keep / remove` 和理由。
- [ ] 压缩比统计，目标显示不超过 30%。
- [ ] RAG 问答，回答带教材、章节、页码引用。
- [ ] 模拟教师反馈，能修改至少一条整合决策并更新图谱或决策列表。
- [ ] README、需求分析、系统设计、Agent 架构说明、整合报告。
- [ ] Public GitHub 仓库 + 公网部署链接。

### 高收益加分项

只有 MVP 跑通后再做。

- [ ] 用 7 本医学教材跑出一份整合报告。
- [ ] 图谱搜索、高亮、频次颜色/大小映射。
- [ ] Few-shot prompt 示例写入 Agent 架构说明。
- [ ] 简单 RAG Benchmark：10-20 个测试问题、引用命中率、响应时间。
- [ ] 教学完整性护栏：展示合并后前置依赖链路是否断裂。

### 明确放弃项

时间不够时不碰。

- [ ] 复杂多 Agent 协作系统，除非只在文档里作为设计讨论。
- [ ] DOCX/Excel 解析，除非 P0 已稳定。
- [ ] 多视图图谱、桑基图、矩阵热力图。
- [ ] Docker 一键部署，除非部署已经稳定。
- [ ] 完整实时处理 824MB 教材；优先保证可运行闭环和可解释流程。

## 取舍规则

- 能展示真实体验的功能优先。
- 能提升提交可信度的 README、部署、错误提示优先。
- 漂亮但不影响核心评分的内容靠后。
- 需要复杂账号、复杂权限、复杂数据清洗的功能先降级。
