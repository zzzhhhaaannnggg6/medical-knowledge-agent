# 技术报告草稿

## 项目名称

医学教材知识整合智能体

## 一句话介绍

本项目把多本医学教材解析成知识图谱，识别重复和互补知识点，在 30% 压缩目标下输出可解释整合结果，并提供带原文引用的 RAG 问答和模拟教师反馈。

## 核心功能

- 教材解析：支持 PDF/Markdown/TXT，记录章节、页码和正文片段。
- 知识图谱：抽取知识点和关系，用 ECharts Graph 可视化。
- 跨教材整合：生成 `merge / keep / remove` 决策、理由和置信度。
- 压缩审计：展示原始正文、整合后正文、压缩比和 30% 是否达标。
- RAG 问答：返回回答、教材、章节、页码、相关度和原文片段。
- 模拟教师反馈：可修改一条整合决策并记录反馈事件。

## 技术架构

- 前端：React + Vite + ECharts + lucide-react。
- 后端：FastAPI + pydantic + pypdf。
- 数据层：JSON state 文件，保存文档、章节、节点、边、决策、压缩和反馈。
- 部署：前端已通过 GitHub Pages 发布静态 Demo；后端作为本地 FastAPI 服务复现真实 MVP 闭环。

## AI / Agent 使用方式

当前 MVP 使用规则抽取、关键词检索和确定性整合流程跑通闭环。Agent 设计上分为教材解析、知识抽取、图谱构建、跨教材整合、压缩审计、RAG 问答和教师反馈七个阶段。

如果比赛要求必须使用模型 API 或 ModelScope，后续可以把知识抽取、合并理由和 RAG 回答生成替换为模型调用；当前文档不虚构尚未接入的模型服务。

## 前端教师四问

前端新增顶部“教师四问”评审栏，围绕老师看项目时最关心的四个判断点组织演示：

| 四问 | 系统回应 |
| --- | --- |
| 为什么合并？ | 整合决策区展示 `merge / keep / remove`、理由、置信度和模拟教师反馈后的变化。 |
| 来源在哪？ | 节点详情、教材列表和 RAG 引用都保留教材、章节、页码。 |
| 压缩≤30%？ | 压缩审计区展示原始正文 65535 字符、整合后 3184 字符、压缩比 4.86%。 |
| 问答有引用？ | RAG 问答返回教材、章节、页码、相关度和原文片段；当前测试问题返回 2 条引用。 |

该设计把评分点前置到界面首屏，帮助评委快速看到合并依据、来源追溯、压缩约束和问答可信度。

## 运行方式

前端：

```bash
cd frontend
npm ci
npm run dev
```

后端：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r backend/requirements.txt
uvicorn app.main:app --reload --app-dir backend
```

烟测：

```bash
BASE_URL=http://127.0.0.1:8001 ./backend/scripts/smoke_test.sh
```

## 已知限制

- 当前 smoke test 只覆盖 2 本教材 MVP 页窗，不代表 7 本教材全量处理。
- RAG 引用已经返回来源，但片段仍含目录噪声，需要继续清洗。
- 公网链接当前是 GitHub Pages 静态 Demo；真实教材解析、RAG 和反馈 API 通过本地 FastAPI 后端复现。
- 当前反馈为模拟教师反馈，不是实际教师在线评审。
