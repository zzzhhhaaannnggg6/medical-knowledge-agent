# 系统设计

## 1. 总体架构

项目采用前后端分离结构：

- 前端：React + Vite + ECharts，展示教材列表、解析状态、知识图谱、整合决策、RAG 问答和教师反馈。
- 后端：FastAPI，负责文件解析、知识抽取、图谱数据生成、跨教材整合、RAG 检索和反馈修改。
- 图谱：ECharts Graph，优先保证缩放、拖拽、点击详情、来源区分。
- 存储：当前后端使用 JSON state 文件保存教材、章节、节点、边、整合决策和反馈记录。
- 模型服务：MVP 当前为规则和关键词检索闭环；模型 API 或 ModelScope 是否强制使用仍待 Root/WeChat 根据官方来源确认。

## 3. 核心模块

### 3.1 文件解析模块

输入 PDF/MD/TXT，输出教材元信息和章节片段。章节片段需要保留页码或页范围，后续图谱和 RAG 都依赖该来源信息。

当前后端提供：

- `POST /api/demo/load`：读取本地默认 demo 教材。
- `POST /api/documents`：上传或按路径解析 PDF/MD/TXT。

### 3.2 知识抽取模块

对章节片段抽取结构化知识点。MVP 字段建议：

```json
{
  "id": "node_001",
  "name": "炎症",
  "type": "concept",
  "definition": "从教材片段中抽取或生成的解释",
  "source_book": "05_病理学.pdf",
  "chapter": "章节名",
  "page_range": "页码"
}
```

### 3.3 图谱模块

节点表示知识点，边表示知识关系。MVP 关系类型：

- `contains`：章节或主题包含知识点。
- `prerequisite`：学习某知识点前需要掌握另一知识点。
- `related`：两个知识点存在相关性但不是包含或前置关系。

### 3.4 跨教材整合模块

跨教材整合模块对相似知识点生成 `merge`、`keep`、`remove` 决策。每条决策应包含动作、来源节点、结果节点、理由和置信度。

### 3.5 压缩审计模块

压缩审计模块计算：

```text
compression_ratio = integrated_text_chars / original_text_chars
```

前端展示原始字数、整合后字数、压缩比和是否达成 `<= 30%`。如果未达标，显示“未达标”和超出原因。

### 3.6 RAG 模块

RAG 模块分为检索和生成两步：先从教材片段中找出相关 chunk，再基于 chunk 生成回答并附引用。没有可靠依据时返回固定未命中文案。

### 3.7 反馈模块

反馈模块接收模拟教师反馈，更新整合决策。当前后端支持 `keep`、`remove`、`merge`、`split` 四类动作。

## 4. 已实现后端接口

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| GET | `/health` | 健康检查 |
| POST | `/api/demo/load` | 加载本地 demo 教材 |
| POST | `/api/documents` | 上传或按路径解析教材 |
| GET | `/api/state` | 获取当前图谱、决策、压缩与引用状态 |
| GET | `/api/dashboard` | 获取前端主面板所需聚合数据 |
| POST | `/api/rag/query` | 提交问题并返回带引用答案 |
| POST | `/api/feedback` | 提交模拟教师反馈并更新决策 |

当前前端主面板接口已对齐到 `/api/dashboard`，`/api/state` 可继续作为调试和完整状态查看接口。

## 5. 前端页面结构

- 教材区：显示教材文件、解析状态、章节数量。
- 图谱区：显示 ECharts Graph，支持节点点击详情。
- 整合区：显示 `merge / keep / remove` 决策、理由、置信度、压缩比。
- 问答区：输入问题，展示答案与引用。
- 反馈区：选择一条决策，提交模拟教师反馈。

## 6. 数据一致性

- 图谱节点必须能追溯到教材、章节和页码。
- 整合决策必须保留原节点和结果节点。
- RAG 引用必须来自原始教材片段，不使用无来源文本。
- 反馈修改必须记录修改前后状态，避免无法审计。

## 7. 部署与安全

- GitHub 仓库必须是 Public。
- 部署链接必须公网可访问，不能是 localhost。
- `.env`、token、cookie、私钥、教材 PDF、解析缓存不提交。
- 若后端部署失败，前端至少保留可展示 Demo 数据和 API 说明。

## 8. 已知限制

- 当前 smoke test 只覆盖 2 本教材的 MVP 页窗，不代表 7 本教材全量处理。
- RAG 引用链路已返回来源，但片段仍存在目录噪声，需要继续清洗。
- 公网部署前仍需确认后端 demo 数据不依赖本机教材路径。
